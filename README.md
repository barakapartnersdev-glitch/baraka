# منصة شركاء البركة — BarakaPartners (MVP)

منصة استثمارية وسيطة تربط أصحاب المشاريع بالمستثمرين عبر حوكمة كاملة للفرص: من بيانات المصدر السرّية حتى النشر العام، مع حماية الهوية و اتفاقية عدم الإفصاح وعدم الالتفاف (NCNDA).

مبنية على **Next.js 16 (App Router) + React 19 + TypeScript + Tailwind + PostgreSQL (عبر Prisma)**، عربية بالكامل (RTL) مع دعم الإنجليزية.

## الحالة: MVP مكتمل وظيفياً — جاهز لمرحلة ما قبل النشر

### البوابات الثلاث (كلها تعمل وموصولة بالبيانات)

- **لوحة الإدارة** `/admin`
  - مؤشرات الصفحة الرئيسية (قيد المراجعة، طلبات جديدة، نواقص، حسابات تنتظر توثيق).
  - قائمة الفرص بحالاتها التسع ونطاق الاستثمار.
  - صفحة الفرصة الداخلية: بيانات المصدر + صياغة النسخ الثلاث للنشر + إدارة النواقص + الملفات + المستثمرون المهتمون + تغيير الحالة.
  - طلبات الاهتمام و NCNDA، وإدارة المستخدمين (اعتماد/إيقاف الحسابات).
- **بوابة صاحب المشروع** `/owner`: إنشاء فرصة، تحرير بيانات المصدر، رفع الملفات، متابعة الحالة والنواقص.
- **بوابة المستثمر** `/investor`: تصفّح الفرص المنشورة، إبداء الاهتمام، توقيع NCNDA إلكترونياً، الوصول لتفاصيل ما بعد التوقيع.
- **الموقع العام**: الرئيسية، الفرص، عن المنصة، كيف تعمل، تواصل.

### القدرات الأساسية

- **مصادقة فعلية**: جلسات JWT موقّعة في كوكي HttpOnly (jose) + تجزئة كلمات المرور (bcryptjs)، وحماية مزدوجة عبر `middleware` على الحافة و`requireRole` داخل الصفحات.
- **حوكمة الفرص**: تسع حالات من `DRAFT_SOURCE` حتى `ARCHIVED`، مع ثلاث نسخ نشر (عامة / مستثمر مسجّل / ما بعد NCNDA) — بيانات المصدر لا تُنشر أبداً كما هي.
- **NCNDA**: نص قانوني عربي + توليد وثيقة HTML موقّعة تُحفظ كأثر دائم.
- **الملفات الآمنة**: تخزين متوافق مع S3 (Cloudflare R2 / AWS / MinIO) مع بديل محلي للتطوير، ومستويات إتاحة (للإدارة فقط / بعد NCNDA)، **وعلامة مائية على PDF** للمستثمر الموقّع.
- **الإشعارات**: داخل المنصة دائماً + بريد إلكتروني عبر Resend (اختياري).
- **ثنائية اللغة**: عربي/إنجليزي مع تبديل لغة في الرأس.
- **سجل النشاط (التدقيق)**: تسجيل الإجراءات الإدارية الحسّاسة.

## متطلبات التشغيل

- Node.js 18 أو أحدث (يُفضّل 20+).
- قاعدة بيانات PostgreSQL قيد التشغيل.

## خطوات التشغيل المحلي

```bash
# 1) ثبّت الاعتماديات
npm install

# 2) جهّز ملف البيئة وعدّل القيم
cp .env.example .env
#   عدّل DATABASE_URL و SESSION_SECRET (32 حرفاً فأكثر) على الأقل

# 3) ولّد عميل Prisma وأنشئ الجداول
npm run db:generate
npm run db:push

# 4) عبّئ البيانات التجريبية (تطوير فقط — تحذف كل البيانات!)
npm run db:seed

# 5) شغّل خادم التطوير
npm run dev
```

ثم افتح:

- الصفحة الرئيسية: http://localhost:3000
- لوحة الإدارة: http://localhost:3000/admin

**بيانات الدخول التجريبية** (كلمة المرور موحّدة `Baraka@123`):
`admin@baraka.example` / `owner1@baraka.example` / `investor1@baraka.example`

## النشر على الإنتاج

راجع [`DEPLOY-VERCEL.md`](DEPLOY-VERCEL.md) لدليل كامل (Vercel + Neon + Cloudflare R2 + Resend).

أهم نقطتين قبل أول نشر:

1. **أنشئ هجرات Prisma وادفعها إلى Git** (مطلوبة لتطبيق المخطط على Vercel):
   ```bash
   npx prisma migrate dev --name init
   ```
2. **أنشئ أول حساب مدير بأمان** (دون حذف بيانات) عبر السكربت المخصّص للإنتاج:
   ```bash
   ADMIN_EMAIL=you@domain.com ADMIN_PASSWORD="سرّ-قوي" ADMIN_NAME="اسمك" npm run db:seed:admin
   ```
   > لا تشغّل `npm run db:seed` على الإنتاج — فهو يحذف كل البيانات.

## بنية المشروع

```
prisma/
  schema.prisma     مخطط قاعدة البيانات (8 نماذج)
  seed.mjs          بيانات تجريبية للتطوير (تحذف كل شيء)
  seed-admin.mjs    إنشاء حساب مدير آمن للإنتاج
src/
  middleware.ts     حماية الحافة للمسارات المحمية
  app/
    admin/          لوحة الإدارة
    owner/          بوابة صاحب المشروع
    investor/       بوابة المستثمر
    (public)        الرئيسية، الفرص، عن، كيف تعمل، تواصل
    login/ register/  المصادقة والتسجيل
    api/            نقاط API (الملفات، NCNDA، seed)
  components/        مكونات مشتركة (الشريط الجانبي، الرأس، الشارات، جرس الإشعارات...)
  lib/
    auth.ts         الجلسات والصلاحيات
    prisma.ts       اتصال قاعدة البيانات
    storage.ts      تخزين S3/محلي
    files.ts        منطق الملفات والإتاحة
    watermark.ts    علامة مائية على PDF
    ncnda.ts        نص الاتفاقية وتوليد الوثيقة
    notify.ts       الإشعارات
    email.ts        البريد (Resend)
    i18n.ts         جداول الترجمة عربي/إنجليزي
    states.ts       تعريفات الحالات بالعربية
    audit.ts        سجل النشاط
    opportunity.ts  منطق الفرص
    account.ts      حالة الحساب
```

## ملاحظات أمنية

- لا ترفع `.env` إلى Git (مستثنى في `.gitignore`).
- التسجيل الذاتي ينشئ **مستثمراً أو صاحب مشروع فقط** — لا مدير. أول مدير يُنشأ عبر `db:seed:admin`.
- على Vercel لا يوجد قرص دائم، لذا **تخزين R2 إلزامي** لميزة الملفات وعلامة NCNDA المائية.
